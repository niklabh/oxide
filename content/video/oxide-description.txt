Oxide is a browser, but it does not load web pages like other browsers. There is no HTML, no JavaScript, no DOM, no thirty years of accumulated quirks; instead, Oxide runs a binary WebAssembly module — and runs it directly. Those programs are written in Rust, and dropped into a sealed sandbox the moment they arrive, which means the app on your screen has no way to read your files, peek at your environment, open a network socket, the camera, or the microphone, unless the browser itself has explicitly handed it that capability. We built the whole thing in Rust on top of wasmtime for execution, and we gave the developer about one hundred and fifty host functions to talk to — a finite, documented surface that covers everything a real app actually needs: a canvas, GPU compute and rendering, audio in and out, video decode through FFMEPG, WebRTC, WebSockets, MIDI, timers, fetch, persistent storage, and a handful of input primitives — and absolutely nothing else, because if we did not register it, it does not exist as far as the developer is concerned. Every module also runs under fuel-metered execution and a hard 256-megabyte memory ceiling, so a runaway loop is not a security incident. On top of this base browser we built Claude 4.7 powered Oxide Forge, which turns the whole thing inside out: instead of writing Rust to make an Oxide app, you open a page called oxide forge, type what you want in plain English — for example, "give me Conway's Game of Life on an eighty by sixty grid" — and Claude Opus 4.7 writes the Rust for you, live, streaming into a side panel as it goes. Forge uses Claude's agent skills system to assemble a custom agent for Oxide — packaging the SDK catalog, idiomatic patterns, and a dozen runnable recipes as a tight, versioned skill — so the model writes against the exact API surface that exists today, not whatever it remembers from training; the code lands in a blank project, it gets compiled, and your app is just running, in the same airtight sandbox as every other Oxide app. If the build fails, Forge pipes the compiler errors straight back to Claude and tries again, so most transient mistakes correct themselves before you see them; and if you want to change something, you do not edit code, you just type another sentence, and Forge keeps the previous version in context, asks Claude to rewrite it. The project can be extended using claude code as well. The result is a browser that does not just run apps but writes them, where the gap between an idea and a working, sandboxed, GPU-accelerated app is one English sentence and one cargo build — and where a wrong answer from the model is, at worst, a compile error, never a vulnerability. That last part matters more than it sounds, because the people who are about to start writing software are not the people who have been writing it: in a year or two, the person describing an app to an AI will be a teacher, a student, a small-business owner, a doctor, a researcher — somebody who has never written code and never will, but who knows exactly what they want software to do for them. They will write most of the software of the future, and the worst possible thing we could hand them is a system where the model's output runs with the same authority as the human at the keyboard — full filesystem, full network, root on the machine — because then every prompt becomes a potential supply-chain attack, every hallucination becomes a potential data leak, and one careless paste is one wiped disk or one stolen credential. The honest truth is that you do not need root to build useful software. The guest only ever sees what the host explicitly hands it, the AI only ever generates code against that same narrow surface, and the modern class of AI-era attacks simply has no shape it can take here, because the primitives it would need are not in the sandbox at all. We think this is what AI-built software has to look like to be safe at the scale it is about to be built at: not a smarter model on a machine that hands it everything, but a small, audited, capability-scoped surface where non-technical people can describe whatever they want, the AI can write whatever it wants, and the worst either of them can do is produce an app that refuses to compile. The future of software is sandboxed binary.
